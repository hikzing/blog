发表于 2014-09-21

素数

素数: 大于1的自然数中，除了1和该数本身，无法被其他自然数整除的数

算术基本定理: 任何大于1的自然数都可以表示成素数的乘积

素数的分解: 除了2和3, 任何素数都可以表示为6*n+1或者6*n-1的形式

不难证明: 因为自然数只能表示成6N±1, 6N±2, … , 6N±6 六种形式, 而只有6N+1和6N+5这两种形式才有可能是素数, 所以这个性质是判断一个数是否为素数的必要不充分条件.

如何判定一个数是否是素数?

1. 利用素数的定义, 如果数字n能被除了1和自己以外的数整除, 那么n就不是素数

2. 或者利用上面提到的素数的分解规律

由于此算法本身效率不高, 因而大多数情况下并不会被用到

3. 模糊算法: 费马素性测试 + 卡迈克尔数

若存在数n, 对于x属于[2, n], 如果符合 (x ^ n) % n == x % n, 那么它有可能是素数.

“有可能”的意思是一个数符合了这个表达式, 但是它不是素数.

上面这个算法是在 n < 1000时是不会误判, 而在 n > 1000 时, 是一种模糊算法, 存在误判的可能.

如果想降低误判率, 可以增加用于校验的值和尽可能多的筛选出卡迈克尔数.

之所以用这样的方法, 因为它可怕的运行效率.

素数的生成

普通的方法就不说了, 无非是对数字一个一个的判断, 如果是素数就输出, 不是就忽略.

很显然, 这样做的效率是非常低的, 我们来研究下其它方法.

1. 利用任意素数可以表示成关于6*n ± 1的性质

hexstep()负责生成6n-1, 6n+1序列, isprime()判断生成的序列是否为素数.

2. Eratosthenes筛选法

简单的说, 就是给出要筛数值的范围n, 从最小的素数(2)开始, 依次把2的倍数除去, 然后找出下一个素数(3), 依次把3的倍数除去, 以此类推, 一直到素数的值大于 n 的开方为止.

上面的实现很简单暴力, 但是却太耗空间和时间了.
